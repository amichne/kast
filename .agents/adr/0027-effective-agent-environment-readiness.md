# ADR 0027: Effective agent environment readiness

Status: Accepted

Date: 2026-07-17

## Context

Task readiness previously proved the selected binary, backend, and managed
workspace artifacts independently. It did not identify every Kast skill that
an agent runtime could discover. A stale user-level skill could therefore
teach commands removed from the running CLI while `kast ready --for agent`
still reported success.

Release and local-development installations also have different authority
models. A Homebrew CLI paired with an IDEA plugin workspace must not be
reported as though it were a receipt-bound local headless generation, and a
local generation must not depend on release receipts or plugin metadata.

This record supersedes the readiness and resource-compatibility portions of
ADRs 0002, 0005, and 0006. It extends, rather than replaces, the
local-development authority in ADR 0024 and the release authority in ADR 0023.

## Decision

`kast ready` reports an `agentEnvironment` projection. For `agent` and
`kotlin` targets, that projection is part of the blocking readiness verdict.
It identifies:

- the exact running binary, implementation version, revision, source path,
  and CLI dialect revision;
- the effective backend kind, version, revision, ownership state, and source
  path;
- the installation authority as `local-development`, `macos-homebrew`,
  `managed-local`, or `missing`;
- every discoverable Kast skill candidate with its path, discovery source,
  ownership state, declared dialect revision, compatibility verdict, and a
  direct repair command when incompatible; and
- the effective managed-guidance path, source, ownership state, and repair
  command when it cannot be trusted.

Readiness validates all discoverable Kast skill candidates rather than
inventing a precedence rule. Agent runtimes can surface duplicate skill names
from repository, user, configured Codex-home, legacy Codex-home, or
administrator roots. Any selectable stale candidate can teach the wrong
dialect, so an incompatible candidate blocks unconditional readiness.

### CLI dialect identity

The authored packaged skill and the IDEA-rendered workspace skill declare
`metadata.kast-cli-dialect-revision`. The running binary derives its expected
revision from its embedded packaged skill. A skill is compatible only when it
parses as the `kast` skill and declares the same revision.

Change the revision when the supported public command dialect changes in a
way that makes older guidance unsafe. Update the packaged skill, IDEA renderer,
cross-language contract tests, and stale-skill regression fixture together.
Do not add a decoder or alias solely to make an old revision appear current.

### Resource states and repair

Agent resources use the closed states `missing`, `modified`, `user-owned`,
`managed`, and `foreign`. Manifest-backed resources are managed only when
their recorded outputs verify. A plugin-owned skill is managed only when its
exact generated bytes match the trusted workspace metadata. Plugin-owned
guidance is managed only when its exact generated `<kast>...</kast>` region is
present; a changed managed artifact is modified, a plugin-owned guidance file
without the region is foreign, and an unowned context file remains user-owned.

Readiness never repairs or replaces content. It may return a shell-safe direct
repair command. User-level incompatible skills are quarantined with a
recoverable `mv` command, plugin-owned resources are refreshed by reopening
the exact IDEA project, managed shell layouts use `kast setup`, and local
generations use `./gradlew refreshDevelopmentLocal`. The operator chooses
whether to execute the command.

### Authority-specific evidence

Local-development readiness consumes the strict local receipt and its
checksummed binary, headless backend, skill, and guidance. It does not require
Homebrew or IDEA workspace metadata. Release-installed macOS readiness
consumes the Homebrew binary receipt and exact plugin workspace metadata.
Managed shell installations consume the install manifest, backend paths, and
recorded resource checksums.
